OpenClaw autonomy operating decisions

Purpose
- Rebuild awesome-openclaw automation using the same high-quality governance shape as awesome-opensource-ai, but with OpenClaw-specific curation rules.

Core model
- Keep only 2 loops: research loop and PR review loop.
- Remove issue-management and staleness automation from current unattended scope.
- Trusted agent GitHub identity is exactly: alvinreal.

Research loop decisions
- Use one global research loop across the whole README, not per-category rotation.
- Each run may open at most 1 PR total.
- Prefer single-category PRs even though discovery is global.
- If an open trusted-agent research PR already exists, do not open another.
- Skip ambiguous candidates.
- Skip projects already proposed by an open community PR when the overlap is exact.

Curation criteria
- Clear OpenClaw relevance is required.
- Candidate must have a working repository, page, or documentation.
- Candidate should be actively maintained or have durable reference value.
- Candidate must fit an existing category.
- Prefer canonical sources over mirrors, forks, or duplicates.

Auto-merge policy
- Auto-merge is allowed, but narrowly.
- Only trusted-agent PRs may auto-merge.
- Auto-merge scope is limited to:
  - README.md only
  - single-category only
  - non-structural changes only
  - mechanically safe changes only
- Initial safe lane includes:
  - broken-link fixes or removals
  - deduplication
  - canonical-source replacement
  - obvious category corrections
  - official OpenClaw resource additions or updates
- Third-party additions are valid for automation to propose, but default to human review unless they are clearly within an explicitly approved safe lane later.

PR review decisions
- Process agent PRs first, then community PRs.
- Skip all draft PRs.
- Community PRs are never auto-merged.
- Community PRs already reviewed are skipped until meaningful new activity arrives.
- Agent PRs are re-evaluated every review run.
- Governance/process changes never auto-merge.

Mechanical checks before merging an agent PR
- Diff touches README.md only.
- Diff stays within one category.
- Diff is non-structural.
- Links resolve.
- GitHub badge format remains correct where used.
- Entry is not duplicated elsewhere in README unless duplicate removal is the purpose.
- Claims of being official are directly verifiable.

Labels
- Keep only these canonical labels:
  - agent:reviewed
  - agent:approved
  - agent:changes-requested
  - agent:rejected
  - needs-human

Migration intent
- OpenClaw should share the same governance shape as awesome-opensource-ai.
- OpenClaw should not copy opensource-ai star/activity/license thresholds.
- OpenClaw should not copy per-category rotation.
